## FAQ

<details open>
  <summary>What is the difference between bastion and opencode?</summary>
  <p>
    Opencode is the primitive for running an AI coding agent in your dev
    environment. Bastion is a layer built on top of opencode to replicate your
    AI coding agent and dev environment into many isolated instances that can
    operate in parallel.
  </p>
</details>

<details open>
  <summary>Why microVMs and not just use git worktrees?</summary>
  <p>
    Git worktrees isolate source code. They do not isolate everything an agent
    touches while working: ports, background processes, package installs,
    databases, temporary files, credentials, and local services. Bastion gives
    each opencode agent its own runtime so agents can build, test, and modify
    code without colliding with one another.
  </p>
</details>

<details open>
  <summary>Can I use this with other coding agents?</summary>
  <p>
    Bastion is built for opencode. On top of being a top tier harness, opencode
    also has an excellent developer ecosystem that allows for powerful open
    source extensions. This allows bastion to focus efforts on being the best in
    class orchestration platform for the open source community.
  </p>
</details>
